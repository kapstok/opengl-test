Om je kubus een Lambert-achtige shader te geven, moet je een verlichtingsmodel implementeren dat diffuse verlichting simuleert. Dit is een basaal verlichtingsmodel waarin het licht diffuus over het oppervlak verspreidt. Lambert-verlichting is gebaseerd op de hoek tussen het licht en het oppervlak en zorgt ervoor dat een oppervlak helderder is wanneer het direct door het licht wordt beschenen, en donkerder wordt naarmate het zich verder van de lichtbron afwendt.

Lambert Verlichtingsmodel

De basisformule voor Lambert-verlichting is als volgt:

Idiffuse=Ilight⋅max⁡(0,N⋅L)I_{\text{diffuse}} = I_{\text{light}} \cdot \max(0, \mathbf{N} \cdot \mathbf{L})Idiffuse​=Ilight​⋅max(0,N⋅L)

Waarbij:

  • IdiffuseI_{\text{diffuse}}Idiffuse​ de diffuse lichtintensiteit is.
  • IlightI_{\text{light}}Ilight​ de intensiteit van het licht (de kleur van het licht).
  • N\mathbf{N}N de normale vector van het oppervlak.
  • L\mathbf{L}L de genormaliseerde vector van de lichtbron naar het oppervlak.
  • N⋅L\mathbf{N} \cdot \mathbf{L}N⋅L de dot-product is tussen de normale vector en de lichtvector.

Stappen om een Lambert-verlichting toe te voegen:

  1. Voeg normaalvectoren toe aan je vertexdata.
  2. Schrijf een vertex shader die normaalvectoren doorgeeft aan de fragment shader.
  3. Schrijf een fragment shader die de diffuse Lambert-verlichting berekent.
  4. Bereken de lichtvector in wereld- of oogruimte.

1. Normaalvectoren toevoegen aan de vertexdata:

Elke vertex moet een normale vector hebben, omdat deze nodig is om de lichtinval te berekenen. De normale vector is een vector die loodrecht op het oppervlak staat.

Hier is een aangepaste versie van je vertex array met normaalvectoren toegevoegd:

2. Vertex Shader aanpassen:

Voeg een extra attribuut toe voor de normaalvectoren en bereken de richting van het licht in de vertex shader.

glsl
#version 330 core
layout(location = 0) in vec3 aPos;    // Positie van de vertex
layout(location = 1) in vec3 aNormal; // Normaalvector van de vertex

out vec3 FragPos;   // Wordt doorgegeven aan de fragment shader
out vec3 Normal;    // Wordt doorgegeven aan de fragment shader

uniform mat4 model;
uniform mat4 view;
uniform mat4 projection;

void main()
{
    // Bereken positie van het fragment in de wereldruimte
    FragPos = vec3(model * vec4(aPos, 1.0));
    // Normaalvector in wereldruimte
    Normal = mat3(transpose(inverse(model))) * aNormal;  
    
    gl_Position = projection * view * vec4(FragPos, 1.0);
}

3. Fragment Shader voor Lambert-verlichting:

Bereken de diffuse verlichting in de fragment shader met behulp van de normale en lichtvector.

glsl
#version 330 core
out vec4 FragColor;

in vec3 FragPos;    // Gekregen van de vertex shader
in vec3 Normal;     // Gekregen van de vertex shader

// Uniformen voor de lichtbron en objectkleur
uniform vec3 lightPos;     // Positie van de lichtbron
uniform vec3 lightColor;   // Kleur van het licht
uniform vec3 objectColor;  // Kleur van het object

void main()
{
    // Bereken de richting van het licht
    vec3 norm = normalize(Normal);  // Normaliseer de normaalvector
    vec3 lightDir = normalize(lightPos - FragPos);  // Richting naar het licht

    // Diffuse Lambert reflectie
    float diff = max(dot(norm, lightDir), 0.0);
    vec3 diffuse = diff * lightColor;

    // Bereken het eindresultaat door diffuse licht toe te passen op de objectkleur
    vec3 result = diffuse * objectColor;
    FragColor = vec4(result, 1.0);
}

4. Instellen van de vertex-attributen in OpenGL:

Zorg ervoor dat je zowel de positie als de normaalvector doorgeeft aan OpenGL:

cpp
// Vertex positie attribuut
glVertexAttribPointer(0, 3, GL_FLOAT, GL_FALSE, 6 * sizeof(float), (void*)0);
glEnableVertexAttribArray(0);

// Normaalvector attribuut
glVertexAttribPointer(1, 3, GL_FLOAT, GL_FALSE, 6 * sizeof(float), (void*)(3 * sizeof(float)));
glEnableVertexAttribArray(1);

5. Verzenden van uniformen naar de shaders:

Vergeet niet de uniformen door te sturen naar de shaders. Bijvoorbeeld voor de lichtbron en het object:

cpp
// Lichtbron positie
glm::vec3 lightPos(1.2f, 1.0f, 2.0f);
glUniform3fv(glGetUniformLocation(shaderProg, "lightPos"), 1, glm::value_ptr(lightPos));

// Lichtkleur
glm::vec3 lightColor(1.0f, 1.0f, 1.0f); // Wit licht
glUniform3fv(glGetUniformLocation(shaderProg, "lightColor"), 1, glm::value_ptr(lightColor));

// Objectkleur
glm::vec3 objectColor(1.0f, 0.5f, 0.31f); // Oranje object
glUniform3fv(glGetUniformLocation(shaderProg, "objectColor"), 1, glm::value_ptr(objectColor));

Resultaat:

Met deze aanpassingen zou je een kubus moeten zien met een Lambert-achtige diffuse verlichting die afhankelijk is van de lichtbron en de oriëntatie van het oppervlak. Dit geeft je object een realistischere verlichte uitstraling.
